# Reglas DQ en consumo (dm/reportes)

Validaciones previas a publicar productos de datos.

Incluye:
- Contratos de salida (columnas/formatos obligatorios) y checks de completitud por periodo.
- Coherencia entre datamarts y sus fuentes core/stage para fechas y agregaciones.
- Comparaciones contra periodos anteriores o fuentes externas cuando aplique.
- Reglas de acceso/sensibilidad asociadas al producto y como se auditan.

Relaciona estos controles con `../../metadata/productos/` y con hallazgos documentados mediante `../plantillas/dq_hallazgos.md`.
