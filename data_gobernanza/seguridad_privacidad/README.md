# Seguridad y privacidad

Registra la clasificacion de datos, controles de uso y manejo de informacion sensible.

Contenido minimo:
- Clasificacion por dataset/campo (publico/interno/confidencial/sensible) y responsables.
- Reglas de enmascaramiento o anonimato y los ambientes donde aplican (dev/test/prod).
- Politicas de retencion y eliminacion para cada dominio o producto.
- Controles de acceso y auditoria, incluyendo proceso de aprobacion y registro de excepciones.
- Riesgos o brechas detectadas y plan de mitigacion.

Cruza referencias con `metadata/` y `dq_reglas/` cuando existan controles de privacidad que se validan en calidad.
